# 📖 書籤檢查小幫手 (Bookmark Checker Pro)

> 一款功能強大的 Chrome 擴充套件，幫助您整理、檢測與管理瀏覽器書籤。  
> 支援重複書籤檢測、失效連結掃描、回收站、匯入匯出等完整功能。

---

## ✨ 功能特色

### 🔍 重複書籤檢測
- **三種比對模式**：完整 URL、網域、根網域，依需求選擇精確度
- **智慧建議**：自動標記建議保留的版本（最新或最舊）
- **全選 / 半選聯動**：支援群組內全選、跨群組全選，核取方塊完美呈現 Indeterminate（半選）狀態
- **即時編輯**：可直接編輯書籤名稱與 URL，修改後自動重新檢測重複狀態
- **安全刪除**：刪除時可選擇放入回收站或永久刪除，並提供確認對話框
- **資料夾篩選過濾與隱藏**：提供結果篩選選單，選擇特定資料夾後，畫面將透過高純淨度的 CSS 過濾，僅呈現該資料夾下的重複書籤，其餘無涉項目自動隱藏。
- **全選防呆安全機制**：當過濾至特定資料夾時，「全選建議刪除項目」核取方塊會智慧判定，**只會勾選當前畫面中可見的項目**，絕對不會誤勾隱藏在背後的其他資料夾書籤，確保刪除安全。
- **檢測結果快取**：切換頁籤後再回來，上次的檢測結果會自動還原，無需重新掃描

### 💀 失效連結檢查
- **檢查前資料夾多選篩選**：重構篩選 UI，引入客製化多選下拉面板（`custom-multiselect`）。
  - **層次化縮排**：載入樹狀資料夾時以 Unicode 不斷行空白進行層次化縮排渲染，層級清晰。
  - **即時關鍵字搜尋**：支援下拉選單內即時關鍵字過濾篩選。
  - **一鍵全選 / 清除**：提供快速勾選與清空捷徑。
  - **防冒泡外部關閉**：點擊選單外部自動收合，交互流暢。
- **五層智慧降級探測引擎**：
  1. **HEAD 快速探測**：使用最輕量的 HEAD 請求進行初步檢查
  2. **GET 降級重試**：HEAD 失敗時自動改用 GET 請求
  3. **HTTPS 自動升級**：對 `http://` 網址自動嘗試 `https://` 版本
  4. **手動重新導向攔截**：使用 `redirect: 'manual'` 攔截重新導向，規避 SSL 跟跳錯誤
  5. **底層憑證錯誤偵測**：透過 `chrome.webRequest` API 捕獲瀏覽器底層的 SSL 憑證錯誤（如 `ERR_CERT_AUTHORITY_INVALID`），準確判定伺服器存活狀態
- **錯誤標籤前置強化**：針對檢測出的異常狀態（⏱️ 逾時、❌ 錯誤狀態碼、↪️ 導向），在書籤標題最前方置入**高對比度的實心微型 Badge**，一眼即可判讀錯誤類型，體驗極佳。
- **真實瀏覽器標頭偽裝**：注入擬真 Chrome 瀏覽器 Headers，繞過防爬蟲防火牆
- **可調整並發數**（1-20）與超時時間（3-60 秒）
- **實時進度條**：顯示掃描進度百分比與已完成數量
- **可隨時停止**掃描
- **單一重新檢查**：對個別失效連結進行單獨重新檢測，完成後顯示結果回饋
- **批量重新檢查選取**：勾選多個連結後一鍵批量重檢
- **智慧重新檢查全部**：一鍵重檢所有失效 / 出錯的連結，自動跳過已確認正常的連結
- **檢測結果快取**：切換頁籤後再回來，上次的檢測結果完整保留

### 🎨 二次過濾「高質感客製化單選下拉選單」
- **突破原生 select 限制**：將重複書籤與失效連結結果上方的「按資料夾過濾」選單，全面升級為**客製化單選下拉元件**，完美突破原生 `<option>` 不能單獨上色的技術瓶頸。
- **紅色計數標章**：在下拉資料夾路徑右側，以醒目的**紅色粗體 `(N)`** 標明該資料夾底下的異常數量，層次分明，資訊極易判讀。
- **大寬度與高度縱深自適應**：下拉選單最大寬度統一擴展至 **`480px`**，結合 **`width: max-content;`** 自動撐開，即使資料夾層級再深，也**絕不折行**！最大高度加長至 **`380px`**，一次能展現超過 12 個選項，大幅減少滾動次數。
- **實心不透明主題背景**：採用主題底色變數 `var(--bg-surface)`，不論在深色主題（質感暗藍）還是亮色主題（高雅純白）下，均擁有完美遮擋下方文字的實心背景，可讀性達到 100%。

### 🗑️ 回收站
- **安全刪除機制**：所有刪除操作預設先放入回收站，避免誤刪
- **一鍵復原**：可將書籤恢復至原資料夾位置
- **永久刪除**：確認後可永久移除，不可恢復
- **資料夾分類篩選**：依來源資料夾分類顯示回收站內容，快速找到目標書籤
- **批量操作**：支援全選 / 反選 / 批量復原 / 批量永久刪除
- **自動清理**：可設定保留天數，超過期限的項目自動清除

### 📦 匯入 / 匯出
- **匯出格式**：
  - **JSON 格式**：完整保留書籤樹結構，適合備份
  - **HTML 格式**：Netscape Bookmark File 格式，相容所有主流瀏覽器匯入
- **匯入功能**：
  - 支援拖曳上傳
  - 自動偵測重複書籤，可選擇跳過或覆蓋

### ⚙️ 設定
- **失效連結檢查**：並發數量、請求超時秒數
- **重複書籤檢測**：預設比對模式
- **回收站**：自動清理天數
- **匯入設定**：重複書籤處理方式（跳過 / 覆蓋 / 全部匯入）

## 🚀 安裝步驟

### 在 Chrome 中載入擴充套件

1. 開啟 Chrome，在網址列輸入：
   ```
   chrome://extensions/
   ```

2. 開啟右上角的「**開發人員模式**」（Developer mode）開關

3. 點擊「**載入未封裝項目**」（Load unpacked）

4. 選擇本專案資料夾

5. 確認擴充套件「**書籤檢查小幫手**」出現在清單中

6. 點擊 Chrome 工具列右上角的拼圖圖示 🧩，將擴充套件釘選到工具列

## 🛡️ 技術亮點

### 五層智慧降級探測引擎

針對各種複雜的網路環境（防火牆、防爬蟲、SSL 憑證不完整等），`background.js` 實作了五層逐級降級的探測策略，確保最大化的連結存活判定準確率：

```
第一層 HEAD (follow)
    ↓ 失敗
第二層 GET (follow)
    ↓ 失敗
第三層 HTTPS GET (follow) — 僅 http:// 網址
    ↓ 失敗
第四層 GET (manual redirect) — 攔截重新導向
    ↓ 失敗
第五層 webRequest 底層憑證偵測 — 穿透 fetch 黑盒
```

### 深度遞迴多資料夾匹配算法
重構 `startCheck()` 邏輯，在檢查前可自由多選不同的目標資料夾。前端採用深度遞迴匹配算法，對書籤在樹狀階層中的父級節點進行全面回溯，能夠精確過濾出多個選定資料夾底下的所有子資料夾書籤進行並發檢查。

### 高質感客製化單選下拉選單系統
為了在選單中為異常數量塗上醒目的紅色，我們採用了自定義的 Custom Dropdown 機制。藉由 Flex 彈性排版、`width: max-content;` 自適應撐寬與 `max-width: 480px` 限制，成功實現了在大資料夾層級下「長文字絕不折行、右側紅色計數標記高精度靠右對齊」的優雅視覺排版。背景則綁定了實心背景與對稱的主題 Hover 態動效，完美契合了亮色/暗色主題切換。

### 檢測結果本地快取

重複書籤與失效連結的檢測結果會自動儲存至 `chrome.storage.local`，切換頁籤後再回來能瞬間還原上次的檢測結果，無需重新掃描。快取載入時會自動與當前書籤樹對齊，過濾已被外部刪除的項目。

## ⚠️ 注意事項

- 部分網站可能因嚴格的 CORS 政策或複雜的防火牆機制，即使五層探測引擎也難以 100% 準確判定。若發現誤判，可手動點擊連結確認。
- 擴充套件需要「讀取所有網站資料」的權限（用於失效連結檢查），這是正常的。
- 回收站資料儲存在 `chrome.storage.local`，清除瀏覽器資料時可能一併刪除。
- 更新 `background.js` 或頁面文件後，需至 `chrome://extensions/` 點擊擴充套件的重新整理按鈕以載入最新版本。
